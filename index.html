TOPIK REACT.JS

PENGERTIAN REACT.JS

React JS merupakan sebuah pustaka javascript yang dibuat oleh facebook. React JS bersifat composable user interface yang berarti pengguna bisa membuat berbagai UI dan nantinya bisa dibagi dalam beberapa komponen dan juga bersifat reusable component yang berarti komponennya bisa digunakan kembali tanpa harus membangun kembali dari awal lagi, React JS bisa digunakan untuk membangun antarmuka pengguna yang khusus untuk tampilan aplikasi dalam satu halaman yang menangani tampilan pada web dan aplikasi seluler

SEJARAH REACT.JS

React resmi dirilis pada tahun 2013, React JS dibuat oleh Jordan Walke. React JS ini telah menjadi salah satu pustaka UI terpenting dalam Javascript, dan telah digunakan oleh Facebook, Instagram, Netflix, Spotify, Medium hingga Twitter. Prototipe React JS yang ia rilis pertama kali Fax JS yang terinspirasi dari XHP. Facebook menyadari bahwa aplikasi web milik mereka semakin kompleks, ada banyak bagian yang harus terus menerus di update seperti pada bagian news feed, chat list, chat box, dan lainnya secara realtime.

KELEBIHAN REACT.JS

  • Kelebihan pertama dari React adalah biaya yang lebih hemat jika dibandingkan dengan proses pengembangan yang lainnya.

  • Bisa menghemat waktu lebih banyak untuk proses pengembangannya.

  • Kelebihan selanjutnya adalah mudah untuk dipelajari dan dipergunakan karena menerapkan gaya penulisan deklaratif.

KEKURANGAN REACT.JS

  • Dokumentasi resmi React kurang tertata dengan baik dibandingkan dengan kebiasaan dokumentasi framework lainnya. Tetapi dokumentasi yang disediakan di luar dokumentasi resmi sudah banyak.

  • React fokus pada bagian View dari suatu aplikasi. Jika ingin membangun aplikasi dalam skala besar, kita harus menyertakan beberapa library luar untuk menangani bagian lain dari suatu aplikasi.

  • Untuk membuat routing pada React, kita harus menyertakan file dari luar.

Slide Performa Dari React Vs Angular


  • Angular bisa membuat aplikasi website dari nol.

  • React hanya membangun tampilan menggunakan komponen yang sudah ada.

  • Angular di buat oleh Google, sedangkan React dibuat oleh Facebook.

  • Pada angular menggunakan konsep “JavaScript ke HTML”. Sedangkan untuk React aplikasi hanya dibangun dalam JavaScript.

Slide Performa Dari React Vs Vue

  • Dokumentasi

    • Vue jelas merupakan pemenang dalam hal dokumentasi. Situs web Vue menampilkan kualitas yang sangat baik, deskripsi yang sangat mendetail, ditawarkan dalam berbagai bahasa, dan dokumen serta referensi API-nya secara luas dianggap sebagai yang terbaik di industri. Anda dapat menemukan jawaban yang jelas untuk sejumlah pertanyaan dan masalah di dokumen. Namun, karena komunitas Vue tidak sebesar React, jadi Anda mungkin akan lebih kesulitan mendapatkan jawaban yang tepat untuk pertanyaan yang tidak tercakup dalam dokumentasi. Dokumentasi React jauh dari level Vue, jadi Anda akan lebih sering beralih ke komunitas untuk memecahkan tantangan dan masalah. Namun, React memang memiliki komunitas yang besar dan aktif, dengan banyak pilihan materi pembelajaran.
    • Arsitektur Aplikasi

      • React berbeda dari framework dan library lain, karena tidak memiliki pola arsitektur bawaan. Ini menggunakan arsitektur berbasis komponen, yang memiliki pro dan kontra. React UI dirender oleh komponen yang berfungsi sebagai fungsi dan merespons perubahan data. Jadi, arsitektur internal terdiri dari interaksi konstan antara keadaan komponen dan tindakan pengguna. Fokus Vue pada pendekatan ViewModel dari pola MVVM bekerja dengan baik untuk aplikasi yang lebih besar. Ini menggunakan pengikatan data dua arah untuk menghubungkan View dan Model. Tujuan utama Vue adalah untuk menyediakan lapisan tampilan yang sederhana dan fleksibel, bukan kerangka kerja yang lengkap.

    • Skalabilitas

      • Saat mempertimbangkan penggunaan React vs. Vue untuk aplikasi besar, React memiliki keunggulan, karena skalabilitasnya yang mudah. Karena aplikasi React hanya menggunakan JavaScript, pengembang dapat menggunakan metode organisasi kode tradisional untuk penskalaan yang mudah. Penggunaan kembali komponen meningkatkan skalabilitas React. Meskipun Vue juga dapat diskalakan, berkat berbagai pilihan alat fleksibelnya, Vue lebih sering digunakan dalam aplikasi yang lebih kecil (walaupun ukuran aplikasi tentu saja bergantung pada arsitekturnya).
